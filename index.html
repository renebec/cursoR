RScripts

Trabajar directo en la consola es genial para experimentar, pero tiene un gran problema: es efímero. En cuanto cierras el programa o limpias la consola, todo lo que escribiste desapareció y no hay forma de recuperarlo. Para evitar esto usamos los RScripts, que son como una "receta de cocina" para tus datos. En lugar de cocinar un ingrediente a la vez, el RScript es un archivo de texto donde dejas guardados todos los pasos ordenados de tu análisis; así puedes corregir errores, guardar tu trabajo y volver a ejecutar todo tu código con un solo clic.



Crearemos un RScript dando click en el ícono de crear nuevo archivo que está en la barra de herramientas principal. Seleccionaremos la opción RScript. Sigue las instrucciones del maestro para que este nuevo RScript tenga por nombre "clase1".




💡 Tip de aprendizaje: Agregar comentarios en tu código

Cada línea de código es una instrucción que el programa ejecutará. Todo lo que escribas a la derecha de un signo de gato (#) será interpretado como un comentario y no como un código de instrucciones a ejecutar. Agregar comentarios en tu código es de gran utilidad para explicar qué es lo que tu código hace o pretende hacer en determinada sección o línea.

Tu objetivo: Desmitificar la programación. No vas a "escribir sistemas complejos", vas a manejar una herramienta de apoyo para tu trabajo. Utilizaremos la analogía de un tractor para entender cómo se divide la interfaz de RStudio:

  • El Script (Editor): Es tu volante. Aquí planeas, escribes tus líneas y decides hacia dónde va el tractor antes de arrancar.
  • La Consola: Es el motor. Aquí es donde ocurre la fuerza y el código se ejecuta de verdad al presionar Enter o Run.

🔍 ¿Para qué sirven las funciones str() y summary()?

Imagina que str() es como hacerle una radiografía a tus datos para ver su estructura interna (saber si son textos o números). Por otro lado, summary() es una inspección técnica rápida que te arroja de inmediato el valor mínimo, el promedio y el máximo de toda tu cosecha.

    # Forzar carga de librería dentro del render para evitar errores en Cloud
    library(ggplot2)
    
    # Parámetros
    n <- as.numeric(input$exponente)
    m <- input$m
    b <- input$b
    minX <- input$rangoX[1]
    maxX <- input$rangoX[2]
    minY <- input$rangoY[1]
    maxY <- input$rangoY[2]
    
    # Crear datos basados en el dominio seleccionado
    datos <- data.frame(x = seq(minX, maxX, length.out = 500))
    datos$y <- m * (datos$x ^ n) + b
    
    # CREAR LA ECUACIÓN MATEMÁTICA (Tipografía profesional)
    # bquote permite usar exponentes reales y símbolos matemáticos
    ecuacion <- bquote(italic(y) == .(m) * italic(x)^.(n) + .(b))
    
    # Graficar
    ggplot(datos, aes(x = x, y = y)) +
      geom_line(color = "#2c3e50", size = 1.5) +
      # Ejes de referencia
      geom_hline(yintercept = 0, color = "black", alpha = 0.3) +

      )
  })
}

tabla_resultados <- mi_data %>%
  # Pasamos las 8 variables a columnas de tipo clave-valor
  # (Reemplaza V1:V8 por los nombres de tu primera y última variable si son distintos)
  pivot_longer(cols = V1:V8, names_to = "Variable", values_to = "Valor") %>%
  
  # Agrupamos para hacer el test de forma independiente por cada variable
  group_by(Variable) %>%
  
  # Ejecutamos la prueba t de Welch y extraemos el p-valor automáticamente
  summarise(
    P_Valor_Original = t.test(Valor ~ Tratamiento, var.equal = FALSE)$p.value,
    .groups = "drop"
  ) %>%
  
  # Calculamos el p-valor ajustado (FDR) y definimos si es significativo
  mutate(
    P_Valor_Ajustado = p.adjust(P_Valor_Original, method = "BH"),
    Diferencia_Significativa = if_else(P_Valor_Ajustado < 0.05, "SÍ", "NO")
  )

# 3. Ver todos los resultados
print(tabla_resultados)

  filter(Diferencia_Significativa == "SÍ")
